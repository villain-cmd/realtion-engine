# 手動変更後も継続する運用手順

## 初回

1. 実績CSV、現在設定CSV、商品マスタCSVをアップロードする。
2. 品質BLOCKがないことを確認する。
3. 計算結果を確認する。
4. 増額は `ACCEPT`、異常行は `MODIFY` または `LOCK` に変更する。
5. 楽天入稿CSVと `state_bundle.zip` をダウンロードする。この時点の変更値は `planned_settings` であり、まだ適用済みではない。
6. 媒体へ入稿する。

## 次回

1. 前回の `state_bundle.zip` をアップロードする。
2. 最新の実績CSVと現在設定CSVをアップロードする。
3. アプリは前回の確認済み適用値、未確認の提案値、手動ロック、実績履歴を突合する。
4. 現在設定CSVが前回提案値と一致した行だけ `PLAN_CONFIRMED` とし、効果観測を開始する。
5. 媒体画面で直接変更した値も、確認済み基準との差分から `EXTERNAL_MANUAL_CHANGE` として検知する。
6. 変更後7日未満は観測状態とする。
7. 7日後、最新実績から通常計算へ戻る。
8. 新しい `state_bundle.zip` を必ず保存する。

## 異常時

`ANOMALY_MANUAL_INTERVENTION` は自動出力されない。

- 現状維持: `REJECT`
- 今回だけ変更: `MODIFY`
- 次回以降も固定: `LOCK`
- 停止: `FORCE_STOP`

手動理由を残すと、将来のルール改善・ML学習時に「人が何を異常と見たか」を教師情報として使える。

## state_bundleを失った場合

最新設定CSVがあれば計算は再開できる。ただし以下は失われる。

- 前回との異常比較
- 手動ロック
- 変更後観測期間
- 過去の推奨・却下理由
- 重複入稿検知
