# テキストマイニング・ネットワーク演算仕様

## 1. 目的

単一行のROASランキングでは把握できない次を可視化します。

- どの商品が多くの需要語を持つか
- どのキーワードが複数商品群を橋渡しするか
- 売上・広告費がどの需要クラスタへ集中しているか
- 近いキーワードが別表記のまま分散していないか
- 商品同士が同じ検索需要を取り合っていないか
- 商品名と入札キーワードの文脈関連度が低くないか

## 2. 文字列正規化

キーワードと商品名へ次を適用します。

1. Unicode NFKC正規化
2. 英字小文字化
3. 全角・半角空白の統一
4. 句読点、括弧、記号の区切り化
5. 連続空白の圧縮

この正規化値をネットワークのキーワード同一性に使用します。原文は表示・監査用に保持します。

## 3. 概念語抽出

Janomeを使い、名詞・動詞・形容詞を抽出します。

- 基本形を優先
- 1文字語、数値のみ、汎用ストップ語を除外
- 解析不能時は空白分割と日本語3文字gramへフォールバック

概念語の規模は、キーワード行のクリック・売上等を語へ配分して集計します。

## 4. キーワード類似度

キーワード文字列へ文字2〜4gramのTF-IDFを適用します。

```text
keyword_similarity(i, j)
= cosine(tfidf_char_2_4(keyword_i), tfidf_char_2_4(keyword_j))
```

各キーワードについて上位近傍だけを残し、GUI指定の閾値以上を接続します。これにより、空白有無、表記揺れ、部分一致を吸収します。

## 5. 商品 × キーワードネットワーク

### ノード

- `PRODUCT`: 商品
- `KEYWORD`: 正規化キーワード

### エッジ

商品×キーワード実績1行を1接続として集約します。

```text
edge_value = selected_metric
```

選択可能な主指標:

- 属性売上
- クリック
- 広告費
- 注文
- 期待粗利増分

レイアウト用の重みは規模の対数変換です。

```text
layout_weight
= 0.25 + log1p(edge_value) / log1p(max_edge_value) * 2.75
```

### 商品名とキーワードの関連度

商品名とキーワードを同じ文字n-gram空間へ配置し、ペアのコサイン類似度を算出します。エッジhoverへ表示します。

## 6. 商品類似ネットワーク

商品同士の接続は次の合成値です。

```text
product_similarity
= 0.7 * shared_keyword_jaccard
+ 0.3 * product_name_char_ngram_similarity
```

共有キーワードが多い商品は、需要の重複・競合・クロスセル候補として近く配置されます。

## 7. ネットワーク指標

### Weighted degree / Hub

接続エッジのレイアウト重み合計です。多くの規模ある接続を持つ中心ノードを示します。

### Betweenness / Bridge

重みの逆数を距離として媒介中心性を計算します。複数クラスタをつなぐキーワード・商品を示します。

### Community

Greedy modularityで接続密度の高いノード群を抽出します。ネットワークが切断されている場合は連結成分をフォールバックにします。

## 8. 規模と相関

- Treemap: 商品規模を面積、ROASを色で表示
- Scatter: クリックをx、属性売上をy、広告費を点サイズ、ROASを色で表示
- Correlation: 外れ値耐性を考慮しSpearman順位相関を表示
- Co-occurrence: 概念語同士の共起をコサイン型正規化で表示

## 9. 表示上限

ブラウザ負荷と可読性を守るため、GUIで商品・キーワードの上限を設定します。上限外のデータを削除するのではなく、選択指標の規模順に表示対象を絞ります。判断・出力テーブルは全件を維持します。

## 10. AI APIを使用しない範囲

次はすべてローカルPython処理です。

- 正規化
- 形態素分解
- TF-IDF
- コサイン類似度
- ネットワーク中心性
- コミュニティ抽出
- 相関計算
- 粗利・CPC演算

外部AI API、Embedding API、LLM APIへの送信はありません。
